# Goutatoo Live — Configuration Supabase (5 minutes)

Pour que votre site se modifie en direct (comme ChefLink), suivez ces étapes.

## 1. Récupérer vos clés Supabase
1. Allez sur https://supabase.com → votre projet
2. Menu **Settings (⚙️) → API**
3. Copiez :
   - **Project URL** (ex: `https://abcdxyz.supabase.co`)
   - **anon public** key (la longue clé `eyJ...`)

## 2. Coller les clés dans le fichier
Ouvrez `goutatoo-live.html`, cherchez tout en haut du `<script>` :

```javascript
var SUPABASE_URL = 'PASTE_YOUR_SUPABASE_URL_HERE';
var SUPABASE_KEY = 'PASTE_YOUR_SUPABASE_ANON_KEY_HERE';
var ADMIN_PASSWORD = 'goutatoo2025';   // ← changez ce mot de passe
```

Remplacez les deux premières valeurs par vos clés, et **changez le mot de passe admin**.

## 3. Créer les tables (copier-coller dans Supabase)
Dans Supabase → **SQL Editor** → **New query** → collez ceci → **Run** :

```sql
-- Table des chefs
create table if not exists chefs (
  id bigint generated by default as identity primary key,
  created_at timestamptz default now(),
  sort int default 0,
  name text, position text, restaurant text,
  cuisine jsonb default '[]', philosophy text, biography text,
  portrait text, restaurant_logo text, restaurant_photo text,
  gallery jsonb default '[]', awards jsonb default '[]',
  timeline jsonb default '[]', recipes jsonb default '[]'
);

-- Sponsors
create table if not exists sponsors (
  id bigint generated by default as identity primary key,
  created_at timestamptz default now(),
  sort int default 0,
  name text, url text, image text
);

-- Photos
create table if not exists photos (
  id bigint generated by default as identity primary key,
  created_at timestamptz default now(),
  sort int default 0,
  title text, description text, image text
);

-- Réglages de la page d'accueil (une seule ligne, id=1)
create table if not exists settings (
  id bigint primary key,
  home_title text, home_sub text, home_tagline text,
  hero_image text, site_logo text
);

-- Autoriser la lecture publique + écriture (anon) pour ce site simple
alter table chefs enable row level security;
alter table sponsors enable row level security;
alter table photos enable row level security;
alter table settings enable row level security;

create policy "public read chefs" on chefs for select using (true);
create policy "public write chefs" on chefs for all using (true) with check (true);
create policy "public read sponsors" on sponsors for select using (true);
create policy "public write sponsors" on sponsors for all using (true) with check (true);
create policy "public read photos" on photos for select using (true);
create policy "public write photos" on photos for all using (true) with check (true);
create policy "public read settings" on settings for select using (true);
create policy "public write settings" on settings for all using (true) with check (true);
```

## 4. Créer le bucket d'images
Dans Supabase → **Storage** → **New bucket** :
- Nom : `goutatoo-images`
- Cochez **Public bucket** ✅
- Create

Puis (SQL Editor) autorisez l'upload public :
```sql
create policy "public upload" on storage.objects for insert with check (bucket_id = 'goutatoo-images');
create policy "public read images" on storage.objects for select using (bucket_id = 'goutatoo-images');
```

## 5. Publier le site
Mettez `goutatoo-live.html` en ligne (GitHub Pages, Netlify, votre hébergeur…).
- **Tout le monde** voit le site et les dernières modifications.
- **Vous** : cliquez le crayon ✏️ en bas à droite → entrez le mot de passe → modifiez → « Enregistrer & publier » → c'est en direct immédiatement.

---

### Sécurité — note importante
Ce montage utilise la clé **anon** avec écriture publique (simple, comme ChefLink). 
N'importe qui qui connaît l'URL de votre base pourrait techniquement écrire dedans.
Pour un usage vitrine c'est acceptable. Si vous voulez une vraie sécurité plus tard,
on pourra passer à Supabase Auth (login email) — dites-le moi.
